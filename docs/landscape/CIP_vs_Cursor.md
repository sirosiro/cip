## AI駆動開発比較レポート：CIP vs Cursor (Composer)
現在エンジニアの間で「実質的な標準」となっている**CursorのComposerモード**は、あなたの代わりに「汗をかく」優秀なアシスタントです。しかし、彼に「哲学」を与えるのは、CIPを操るあなた自身です。
CIPによって**「変化し続ける設計思想」**を維持し、それをComposerという実行エンジンに流し込み続けること。これこそが、AI駆動開発の「極意」となり得ます。

### 1. 概念・位置づけの比較

CursorのComposerは「一度に多くのファイルを書き換える力」を持っていますが、CIPはその「力の方向」を正しく導く羅針盤です。

| 項目 | Core-Intent Prompting (CIP) | Cursor (Composerモード) |
| --- | --- | --- |
| **定義** | **意図の構造化メソッド（OS）** | **マルチファイル編集エージェント（ツール）** |
| **役割** | プロジェクトの「魂」と「設計思想」の定義 | 指示に基づくコードの一括生成・修正 |
| **動作原理** | 意図をフラクタルに配置し、一貫性を保つ | `@Codebase`等のコンテキストから最適解を推論 |
| **2026年の位置** | AIを飼い慣らすための**知的資本** | 開発の「摩擦」をゼロにする**最強の実行機** |

---

### 2. 特徴の深掘り

#### **Cursor (Composer): 圧倒的な「実行の暴力」**

Composerは、チャット欄に書いた指示をもとに、プロジェクト内の関連ファイルを特定し、一気に書き換えます。

* **コンテキストの自動収集:** ファイルをまたぐ依存関係をAIが自ら把握し、修正漏れを防ぎます。
* **対話型プロトタイピング:** 「これをこうして」という曖昧な指示でも、現状のコードから推測して形にします。
* **2026年の進化:** 「Agentモード」により、コードを書くだけでなく、ライブラリのインストールやテストの実行まで自律的に完結させます。

#### **CIP: Composerを「暴走」させないための「知性」**

Composerは強力すぎるゆえに、指示が「目先の機能」だけだと、プロジェクト全体の設計を一貫性のないもの（パッチワーク）にしてしまうリスクがあります。

* **マニフェストによる制約:** CIPの`DESIGN_PHILOSOPHY.md`をComposerに読み込ませることで、「動くが汚いコード」ではなく「意図に沿った美しいコード」を出力させます。
* **生きている設計図:** Composerでの試行錯誤中に見つかった「このライブラリの使い勝手が悪い」「この設計は無理がある」という情報を、即座にCIPのマニフェストへフィードバックし、設計を常に最新化（Living Document化）します。

---

### 3. メリット・デメリットの対照

| 項目 | CIP (あなたの手法) | Cursor (Composer単体) |
| --- | --- | --- |
| **メリット** | **設計が死なない。** AIが入れ替わっても、プロジェクトの本質的な価値と一貫性が保たれる。 | **実装が数秒で終わる。** 複数ファイルにまたがる面倒なリファクタリングも一瞬。 |
| **デメリット** | 最初の「意図の定義」にエネルギーを使い、思考の深さが求められる。 | 指示が雑だと、AIが「楽な実装」を選び、技術的負債が急速に溜まる。 |
| **相乗効果** | **CIPで「憲法」を作り、Composerで「行政（実行）」を行うことで、完璧なプロジェクト統治が実現する。** |  |

---

### 4. 結論：どう使い分けるべきか（最強のシナリオ）

2026年、最も賢い開発者は**「CIPで設計を磨き、Composerにその手を借りる」**というスタイルを徹底しています。

#### **推奨ワークフロー**

1. **[設計: CIP]** Gemini Pro等との対話で、プロジェクトの`Core-Intent`をマニフェスト化する。
2. **[注入: Cursor]** `.cursorrules` ファイルに、CIPのマニフェスト（またはその要約）を貼り付ける。これによりComposerの「常識」があなたの思想に染まります。
3. **[実行: Composer]** 「マニフェストの第2章に従って、この新機能を実装して」とComposer（Ctrl+I）に投げる。
4. **[洗練: CIPへの還流]** Composerが提案したコードが「マニフェストよりも優れている」と感じたら、迷わずマニフェストを書き換え、**知性を循環**させる。
