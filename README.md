# AI協調開発フレームワーク (AI Collaborative Development Framework)

本プロジェクトは、人間とAIが「設計意図（Intent）」を核に協力し、高品質で持続可能なソフトウェアを開発するためのフレームワークです。

## コンセプト
単なるコード補完や生成ではなく、AIを「熟練のアーキテクト」としてパートナーに迎えます。
プロジェクトの憲法となる `ARCHITECTURE_MANIFEST.md` を共同で設計・運用し、「なぜその設計にしたのか」という意図を共有し続けることで、長期的な保守性と一貫性を保証します。

## 実行環境について
本フレームワークは、特定のAIモデルやツールにハードに依存するものではありません。
ただし、**gemini-cli** 上での利用を推奨環境としています。これは、Google Geminiモデル（特にFlash Thinkingなどの推論特化型モデル）が持つ「思考プロセス（Thinking Mode）」と長いコンテキストウィンドウが、アーキテクチャ設計という抽象度の高いタスクに適しているためです。

十分な推論能力とコンテキスト長を持つ他のLLM（Claude 3.5 Sonnet, GPT-4oなど）でも、プロンプトを読み込ませることで利用可能です。

## 使い方 (How to Use)
本フレームワークの核となるのは、AIへの「初期人格形成プロンプト」です。

1. **プロンプトの準備**: [DESIGN_PHILOSOPHY.md](./DESIGN_PHILOSOPHY.md) の全内容をコピーします。
2. **AIの起動**: 任意のAIチャットツール（推奨: gemini-cli）を起動します。
3. **ブートストラップ**: 最初のメッセージとして、コピーした内容を貼り付けて送信します。
4. **協調作業の開始**: AIが内容を理解し、次のアクション（新規設計または既存解析）を提案してくるので、指示に従って対話を進めてください。

## 今後の展望: フラクタル・アーキテクチャ
大規模プロジェクトへの対応として、本フレームワークが生み出すマニフェスト（`ARCHITECTURE_MANIFEST.md`）を**フラクタル（再帰的）に配置する手法**を計画しています。
システム全体のマニフェストの下に、サブシステム、モジュールごとのマニフェストを階層的に配置することで、複雑なドメインでも一貫した設計思想を隅々まで浸透させることを目指します。

## 主要ドキュメント
- [DESIGN_PHILOSOPHY.md](./DESIGN_PHILOSOPHY.md): 本フレームワークの根本思想であり、AIパートナーへのブートストラップ・プロンプトです。
